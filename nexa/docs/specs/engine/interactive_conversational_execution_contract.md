# Interactive Conversational Execution Contract v0.1

## Recommended save path
`docs/specs/engine/interactive_conversational_execution_contract.md`

## 1. Status
Deferred.

## 2. Purpose

This document defines the future contract for interactive or conversational execution flows in Nexa.

## 3. Core Rule

Interactive execution must not blur the distinction between explicit circuit execution, approval boundaries, and conversational back-and-forth.

## 4. Minimum Future Contract Areas
- turn identity inside a run
- pause/resume interaction boundary
- conversational state projection
- artifact/trace continuity across turns
- approval/review semantics during interaction

## 5. Decision

Interactive execution is deferred because it represents a stronger product-character shift than the current productization-first line.
