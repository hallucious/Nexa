# Beginner / Advanced Shell Split v1

## Recommended save path
`docs/specs/ui/workspace_shell/10_beginner_advanced_shell_split.md`

## 1. Purpose

This document defines the two presentation densities of the same product:

- Beginner shell
- Advanced shell

These are not separate applications.

## 2. Core rule

Both shells share:
- same engine truth
- same storage model
- same graph
- same execution history
- same approval boundaries

Only density, simultaneity, and explanatory surface change.

## 3. Beginner shell goals

- reduce fear
- shorten time to first successful loop
- show meaning before mechanics
- minimize simultaneous density

Beginner default should favor:
- graph
- designer input
- validation summary
- run action
- result reading

Beginner should hide or collapse:
- deep trace
- provenance-heavy diff
- advanced filters
- dense metrics walls

## 4. Advanced shell goals

- shorten expert loops
- expose more observability
- expose deeper trace/diff/storage surfaces
- support dense inspection without mode confusion

Advanced default may show:
- more badges
- more panels
- more overlays
- denser bottom dock
- faster drill-down affordances

## 5. Wrong interpretation

Beginner is not fake simplicity.
Advanced is not "real data mode."
Both see the same truth.
Only presentation and density change.
