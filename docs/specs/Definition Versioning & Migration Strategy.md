Hyper-AI Definition Versioning & Migration Strategy v0.1
(정의 파일 버전 관리 및 마이그레이션 전략 – 복붙용 단일 블록)
======================================================================

목적:
Definition Language가 진화하더라도
기존 정의 파일이 깨지지 않도록 보장한다.

----------------------------------------------------------------------
1. 버전 체계

Definition 파일은 두 개의 버전을 가진다:

1.1 format_version
- 정의 파일 포맷 버전
- Hyper-AI 언어 구조 자체의 버전
- 예: "0.1.0"

1.2 definition_revision
- 특정 설계도의 개별 수정 버전
- 사용자 정의
- 예: "1.3.2"

----------------------------------------------------------------------
2. 버전 규칙 (SemVer 기반)

MAJOR (X.0.0)
- 구조적 변경
- 기존 정의 파일과 비호환 가능
- 마이그레이션 필요

MINOR (0.X.0)
- 기능 추가
- 하위 호환 유지

PATCH (0.0.X)
- 버그 수정
- 구조 변경 없음

----------------------------------------------------------------------
3. Runtime 호환성 정책

3.1 동일 MAJOR
- 자동 실행 허용

3.2 상위 MINOR
- 하위 정의 파일 실행 허용
- 경고 가능

3.3 상위 MAJOR
- 자동 실행 금지
- 마이그레이션 필수

----------------------------------------------------------------------
4. 마이그레이션 모델

4.1 자동 마이그레이션

- 구조 변경이 명확한 경우 자동 변환
- 예: 필드 이름 변경, 기본값 추가

4.2 반자동 마이그레이션

- 사용자 확인 필요
- 예: 분기 구조 변경

4.3 수동 마이그레이션

- 계약 변경
- 노드 타입 변경
- 의미적 변경

----------------------------------------------------------------------
5. 마이그레이션 원칙

1) 구조 의미를 변경하지 않는다.
2) 암묵적 추측 변환 금지.
3) 실패 시 자동 롤백.
4) 원본 정의 파일 보존.
5) 마이그레이션 로그 기록.

----------------------------------------------------------------------
6. 정의 파일 내 필수 필드

{
  "format": "hyperai.definition",
  "format_version": "0.1.0",
  "definition_revision": "1.0.0"
}

----------------------------------------------------------------------
7. 버전 불일치 처리

7.1 미래 버전 감지
- 현재 Runtime이 이해 못하는 format_version
- 실행 차단
- "UNSUPPORTED_DEFINITION_VERSION" 오류 반환

7.2 과거 버전 감지
- 지원 범위 내면 자동 실행
- 지원 범위 외면 마이그레이션 시도

----------------------------------------------------------------------
8. 정의 파일 안정성 원칙

1) 정의 파일은 불변 아티팩트다.
2) 수정 시 definition_revision 증가.
3) 과거 revision은 항상 재현 가능해야 한다.
4) Runtime은 특정 format_version 범위만 지원한다.
5) 실행 결과는 format_version과 함께 기록된다.

----------------------------------------------------------------------
9. 철학 요약

- 진화는 허용한다.
- 붕괴는 허용하지 않는다.
- 호환성은 명시적으로 관리한다.
- 자동 추측 업그레이드는 금지한다.
