======================================================================
Hyper-AI Runtime Responsibility Specification v0.1
(런타임 책임 범위 정의 – 복붙용 단일 블록)
======================================================================

목적:
정적 검사를 통과한 Definition을
안전하고, 예측 가능하며, 추적 가능하게 실행한다.

----------------------------------------------------------------------
1. 런타임의 역할

Runtime은 다음 5가지를 책임진다:

1) 실행 순서 제어
2) 계약 준수 강제
3) 실패 처리 수행
4) 관측성 제공
5) 실행 격리 및 안전 보장

Runtime은 "구조를 해석하고 실행하는 기계"다.
구조를 변경하지 않는다.

----------------------------------------------------------------------
2. 실행 보장 (Execution Guarantees)

2.1 정의 불변성
- 실행 중 Definition 구조 변경 금지
- 동적 노드 생성 금지 (v0.1 기준)

2.2 계약 강제
- Node 실행 전 입력 스키마 재검증
- Node 실행 후 출력 스키마 검증
- 불일치 시 즉시 실패 처리

2.3 결정성 정책
- temperature 등 설정은 Definition 설정 우선
- Runtime이 임의 변경 금지

----------------------------------------------------------------------
3. 실행 흐름 제어

3.1 Sequential
- start부터 정의된 연결 순서대로 실행

3.2 Conditional
- 분기 조건 평가 후 단일 경로 선택
- 조건 평가 오류 시 즉시 실패

3.3 Parallel
- 병렬 노드 독립 실행
- 병합 전 모든 노드 완료 확인
- 하나라도 실패 시 on_error 정책 적용

3.4 Error Routing
- reason_code 기반 정책 적용
- retry / stop / route 정확히 수행
- 재시도 횟수 초과 시 즉시 중단

----------------------------------------------------------------------
4. 관측성 (Observability)

4.1 로그 단위
- 노드 단위 시작/종료 로그
- 입력/출력 요약 로그
- reason_code 로그

4.2 실행 추적 ID
- 전체 실행에 execution_id 부여
- 각 노드 실행에 trace_id 부여

4.3 구조 추적 가능성
- 어떤 경로가 선택되었는지 기록
- 분기 평가 결과 기록
- 병렬 병합 기록

----------------------------------------------------------------------
5. 안전성 (Safety Boundaries)

5.1 플러그인 격리
- sandbox 정책 준수
- 파일 시스템 접근 제한
- 네트워크 접근 정책 적용

5.2 시간 제한
- 노드 단위 timeout 강제
- 전체 실행 timeout 가능

5.3 리소스 보호
- 무한 재시도 방지
- 병렬 폭주 방지

----------------------------------------------------------------------
6. 실패 모델

모든 실패는 다음 구조로 반환:

{
  "success": false,
  "node_id": "...",
  "reason_code": "...",
  "message": "...",
  "trace_id": "...",
  "execution_id": "..."
}

Runtime은 실패를 숨기지 않는다.
Runtime은 추측 복구하지 않는다.
Runtime은 명시적 정책만 따른다.

----------------------------------------------------------------------
7. Runtime이 하지 않는 것 (Non-Responsibilities)

Runtime은:

- 계약을 자동 수정하지 않는다
- 타입을 자동 변환하지 않는다
- 누락 필드를 추측 채우지 않는다
- 구조를 자동 재작성하지 않는다
- 사용자의 의도를 해석하지 않는다

Runtime은 "명령된 구조만 실행"한다.

----------------------------------------------------------------------
8. 철학 요약

1) 실행은 구조에 종속된다.
2) 계약은 절대적이다.
3) 실패는 명확해야 한다.
4) 암묵적 복구는 금지한다.
5) 관측성은 기본값이다.
