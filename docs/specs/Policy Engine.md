======================================================================
Hyper-AI Policy Engine Specification v0.1
(정책 엔진 분리 설계 – 복붙용 단일 블록)
======================================================================

목적:
보안, 리소스 제약, 실행 제한을 Runtime에서 분리하여
독립적인 Policy Engine으로 관리한다.

Runtime은 실행만 담당하고,
Policy Engine은 "허용 여부"를 판단한다.

----------------------------------------------------------------------
1. 역할 분리 원칙

1.1 Runtime
- 구조 해석
- 실행 순서 제어
- 상태 관리
- 계약 강제

1.2 Policy Engine
- 실행 허용 여부 판단
- 리소스 제한 적용
- 외부 접근 제어
- 비용 제한
- 위험 행위 차단

Runtime은 Policy를 우회할 수 없다.

----------------------------------------------------------------------
2. 정책 적용 단계

정책은 다음 시점에 적용된다:

2.1 Definition Load 시
- 허용되지 않은 plugin 사용 차단
- 금지된 모델 차단

2.2 Node 실행 전
- 실행 허용 여부 검사
- 예상 비용 초과 여부 검사
- sandbox 정책 확인

2.3 Node 실행 후
- 출력 데이터 크기 제한 검사
- 위험 내용 검사 (옵션)

----------------------------------------------------------------------
3. 정책 종류

3.1 Execution Policy
- max_total_runtime_ms
- max_node_runtime_ms
- max_parallel_nodes
- max_retry_per_node

3.2 Resource Policy
- max_total_tokens
- max_total_cost
- max_output_size
- memory_limit (향후 확장)

3.3 Security Policy
- sandbox_plugins (true/false)
- allowed_plugin_list
- allowed_model_list
- network_access (허용/차단)

3.4 Risk Policy (옵션)
- 금지 키워드 검사
- 민감 정보 필터
- 외부 전송 제한

----------------------------------------------------------------------
4. 정책 평가 결과 구조

Policy Engine은 항상 명확한 결과를 반환한다:

{
  "allowed": true | false,
  "reason_code": "...",
  "message": "...",
  "policy_type": "execution|resource|security|risk"
}

Runtime은 allowed=false일 경우 즉시 중단한다.

----------------------------------------------------------------------
5. 정책 우선순위

정책 위반은 다음 순서로 평가된다:

1) Security
2) Resource
3) Execution
4) Risk

가장 높은 위반이 반환된다.

----------------------------------------------------------------------
6. 비용 가드레일 (Cost Guardrail)

- 실행 전 예상 토큰 계산 가능 시 사전 차단
- 실행 중 누적 비용 초과 시 즉시 중단
- partial 결과 허용 여부는 정책에 따름

----------------------------------------------------------------------
7. 확장 가능 구조

Policy Engine은 플러그인 형태로 확장 가능:

- Custom Policy Module 등록 가능
- 조직별 정책 정의 가능
- 환경별 정책 분리 가능 (dev / prod)

----------------------------------------------------------------------
8. 금지 규칙

- Runtime이 정책을 수정하지 않는다.
- 정책 위반을 자동 무시하지 않는다.
- 암묵적 허용 없음.
- 정책 실패를 숨기지 않는다.

----------------------------------------------------------------------
9. 철학 요약

1) 실행과 통제는 분리된다.
2) 정책은 구조 위에 존재한다.
3) 비용은 통제 대상이다.
4) 보안은 선택이 아니다.
5) 정책 위반은 즉시 중단한다.
