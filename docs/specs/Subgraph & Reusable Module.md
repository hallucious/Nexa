Hyper-AI Subgraph & Reusable Module Specification v0.1
(재사용 가능한 서브그래프 설계 – 복붙용 단일 블록)
======================================================================

목적:
복잡한 Node 묶음을 하나의 재사용 가능한 모듈로 정의한다.
조합 복잡도를 낮추고 구조 확장성을 확보한다.

----------------------------------------------------------------------
1. 기본 개념

Subgraph는:
여러 Node + 연결 + 내부 흐름을 포함하는
하나의 "논리적 실행 단위"이다.

Subgraph는 외부에서 보면
하나의 Node처럼 동작해야 한다.

----------------------------------------------------------------------
2. Subgraph의 필수 요소

각 Subgraph는 다음을 가진다:

- module_id (고유 식별자)
- name
- version
- internal_definition (내부 Definition 구조)
- public_contract

----------------------------------------------------------------------
3. Public Contract (외부 계약)

Subgraph는 반드시 외부에 노출되는 계약을 정의한다:

public_contract:
  - in (입력 스키마)
  - out (출력 스키마)
  - fail_codes

외부에서는 내부 Node 구조를 알 필요가 없다.
오직 public_contract만 본다.

----------------------------------------------------------------------
4. 내부 구조

internal_definition은
일반 Definition과 동일한 구조를 가진다:

- nodes
- edges
- flow

단, 내부 Node는 외부에서 직접 접근 불가.

----------------------------------------------------------------------
5. 실행 원칙

5.1 Subgraph 호출 시:

- 외부 입력 → public_contract.in 검증
- 내부 start 노드로 전달
- 내부 실행 완료 후
- 결과 → public_contract.out 검증

5.2 내부 실패 처리:

- 내부 reason_code → public_contract.fail_codes로 매핑
- 내부 실패 코드 직접 노출 금지 (옵션)

----------------------------------------------------------------------
6. 버전 관리

Subgraph는 자체 version을 가진다.

- MAJOR 변경: public_contract 변경
- MINOR 변경: 내부 구조 변경 (외부 계약 유지)
- PATCH 변경: 내부 버그 수정

Subgraph version은 Definition에서 명시적으로 고정(pinned)해야 한다.

----------------------------------------------------------------------
7. 중첩 규칙

- Subgraph 내부에서 또 다른 Subgraph 호출 가능
- 순환 모듈 참조 금지
- 자기 자신 호출 금지 (v0.1 기준)

----------------------------------------------------------------------
8. 검증 규칙

Subgraph 저장 시 반드시 검사:

1) public_contract 존재
2) internal_definition 유효성 검사 통과
3) 내부 start 노드 존재
4) 내부 노드 고립 없음
5) public_contract와 내부 입출력 연결 일관성

----------------------------------------------------------------------
9. 철학 요약

1) Subgraph는 추상화 단위다.
2) 내부 구조는 캡슐화된다.
3) 외부는 계약만 본다.
4) 재사용은 계약 안정성 위에서만 허용된다.
5) 복잡성은 계층화로 통제한다.
