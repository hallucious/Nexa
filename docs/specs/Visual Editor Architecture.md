======================================================================
Hyper-AI Visual Editor Architecture v0.1
(시각적 조합 UI 설계 – 복붙용 단일 블록)
======================================================================

목적:
시각적 조합 UI를 통해 Definition을 생성·수정하되,
JSON 정본(Definition Language)을 단일 진실 소스(Single Source of Truth)로 유지한다.

----------------------------------------------------------------------
1. 핵심 원칙

1) JSON 정본이 유일한 진실이다.
2) UI는 JSON을 생성/수정하는 도구일 뿐이다.
3) UI 상태는 JSON과 항상 동기화되어야 한다.
4) 라운드트립(Visual → JSON → Visual)이 완전해야 한다.

----------------------------------------------------------------------
2. 구조 개요

[User Interaction]
        ↓
[Visual Graph Model]
        ↓ (serialize)
[JSON Definition (정본)]
        ↓ (validate)
[Static Validator]
        ↓
[Runtime Execution]

UI는 Validator를 우회할 수 없다.

----------------------------------------------------------------------
3. Visual Graph Model

UI 내부 표현은 다음 요소를 가진다:

3.1 Node Element
- node_id
- label (name)
- kind
- position (x, y)
- collapsed 상태 여부
- subgraph 여부

3.2 Edge Element
- source_node_id
- target_node_id
- field_mapping
- conditional_flag

3.3 Flow Layer
- branch blocks
- parallel blocks
- error routing blocks

----------------------------------------------------------------------
4. 라운드트립 규칙

4.1 Visual → JSON
- 모든 노드, 연결, 흐름은 JSON으로 직렬화
- UI 전용 정보(position 등)는 별도 metadata에 저장

4.2 JSON → Visual
- JSON을 파싱해 동일 그래프 재구성
- UI 전용 metadata 복원
- 의미 변경 금지

----------------------------------------------------------------------
5. 즉시 검증 (Live Validation)

UI는 다음을 즉시 표시해야 한다:

- 타입 불일치 (빨간 연결선)
- 필수 필드 누락
- 고립 노드
- 순환 구조 오류
- 정책 위반 경고

정적 검사는 항상 백엔드 Validator 기준으로 수행한다.

----------------------------------------------------------------------
6. Subgraph 표현

- Subgraph는 하나의 노드처럼 표시
- 더블클릭 시 내부 그래프 열림
- public_contract는 외부에서만 표시
- 내부 노드는 외부 그래프에서 숨김

----------------------------------------------------------------------
7. UX 철학

1) 구조는 엄격하지만 표현은 단순해야 한다.
2) 사용자가 계약을 직접 작성하지 않아도 되도록
   스키마 기반 자동 입력 폼 제공.
3) 암묵적 연결 금지.
4) 오류는 시각적으로 명확히 표시.
5) 구조 복잡성은 계층화로 해결.

----------------------------------------------------------------------
8. 버전 및 Registry 연동

- 저장 시 Registry에 revision 생성
- 기존 revision 수정 불가
- UI에서 revision 비교(diff) 가능
- format_version 표시

----------------------------------------------------------------------
9. 권한 모델

- 읽기 전용 모드
- 편집 모드
- 승인 필요 모드 (조직 환경)

----------------------------------------------------------------------
10. 확장 가능성

향후 추가 가능:

- 실행 시각화 (실시간 경로 표시)
- 토큰 사용량 시각화
- 실패 통계 히트맵
- 드래그 기반 병렬/분기 생성
- 텍스트 DSL 병행 편집 모드

----------------------------------------------------------------------
11. 금지 규칙

- UI가 JSON 구조를 자동 보정하지 않는다.
- 암묵적 필드 추가 금지.
- 검증 우회 저장 금지.
- 내부 ID 재생성 금지.

----------------------------------------------------------------------
12. 철학 요약

1) JSON이 진실이다.
2) UI는 편의 도구다.
3) 구조는 엄격하다.
4) 검증은 항상 선행된다.
5) 복잡성은 계층화로 통제한다.
