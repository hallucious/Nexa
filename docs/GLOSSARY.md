# Nexa Glossary

This document defines key terminology used throughout the Nexa project.

---

Circuit

A graph structure that defines how nodes are connected and executed.

---

Node

The smallest execution unit in Nexa.

All computation occurs inside nodes.

---

Runtime

The system responsible for executing nodes and managing execution state.

---

Prompt

Instructions provided to an AI model.

---

Provider

A system that connects Nexa to an AI model service.

---

Plugin

An extension module that performs non-AI computation.

---

Artifact

An immutable output produced during execution.

---

Execution Trace

A record of runtime behavior during circuit execution.

---

Working Context

The shared data space used by runtime components.

---

Contract

A rule that defines expected behavior between components.

---

Deterministic Execution

Execution that produces identical results given the same inputs.

---

End of Glossary
